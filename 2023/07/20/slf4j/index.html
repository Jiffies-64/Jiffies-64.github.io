Slf4j笔记

官方网站： https://www.slf4j.org/

概述

Slf4j使用了外观模式。

SLF4J（Simple Logging Facade for Java）是一个为Java应用程序提供日志功能的简单日志门面（Facade）框架。它允许应用程序在运行时使用不同的日志实现，而无需更改应用程序的代码。SLF4J 提供了一个统一的接口，供应用程序与底层的日志系统进行交互，比如常用的日志实现：Logback、Log4j以及java.util.logging（JUL）等。

SLF4J 的使用非常简单，一般包括以下几个步骤：

  • 引入 SLF4J 库：将 SLF4J 的相关库文件添加到项目的依赖中。
  • 编写日志代码：在应用程序中使用 SLF4J 的 API 来记录日志信息，如记录错误、警告、信息等级的日志。
  • 配置日志实现：在项目的配置文件中指定要使用的具体日志实现，例如 Logback 或 Log4j。

源码分析

SLF4J通过LoggerFactory加载日志具体的实现对象。

LoggerFactory在初始化的过程中，会通过performInitialization()方法绑定具体的日志实现。

在绑定具体实现的时候，通过类加载器，加载org/slf4j/impl/StaticLoggerBinder.class

所以，只要是一个日志实现框架，在org.slf4j.impl包中提供一个自己的StaticLoggerBinder类，在其中提供具体日志实现的LoggerFactory就可以被SLF4J所加载
